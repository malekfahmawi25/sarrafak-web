# صرافك

المشروع يستخدم الآن قاعدة بيانات حقيقية من نوع:

```text
PostgreSQL
```

لم يعد الخادم يقرأ الأرصدة أو العمليات من ملف نصي.

## الحماية المضافة

- حفظ الأرقام السرية كبصمات قوية مع قيمة سرية إضافية خارج الكود.
- حفظ بصمات رموز الجلسات فقط داخل قاعدة البيانات.
- استخدام ملف ارتباط محمي لا تستطيع شيفرة الصفحة قراءته.
- انتهاء الجلسة بعد مدة محددة أو بعد عدم الاستخدام.
- إيقاف محاولات الدخول مؤقتاً بعد تكرار الأخطاء.
- تنفيذ السحب والإيداع داخل معاملة مترابطة مع قفل لسجل الحساب.
- منع تنفيذ العملية نفسها مرتين عند تكرار الطلب.
- استخدام استعلامات ذات معاملات لمنع حقن أوامر قاعدة البيانات.
- تقييد مصادر الاتصال وإضافة رؤوس حماية للمتصفح.
- تسجيل الأحداث الحساسة في سجل تدقيق منفصل.

## تنبيه مهم

هذا المشروع أصبح أكثر أماناً، لكنه ليس نظاماً بنكياً معتمداً أو بديلاً عن مراجعة أمنية مستقلة.

الإيداع الذاتي معطل افتراضياً؛ لأن المستخدم لا يجوز أن يضيف رصيداً لنفسه في نظام حقيقي. يمكن تفعيله فقط أثناء العرض التجريبي.

## متطلبات التشغيل المحلي

يلزم تثبيت:

```text
Node.js 24
PostgreSQL
```

أنشئ قاعدة بيانات محلية باسم:

```text
sarrafak
```

انسخ الملف:

```text
.env.example
```

إلى ملف جديد باسم:

```text
.env
```

ثم ضع داخله رابط قاعدة البيانات وقيمتين سريتين مختلفتين، لا تقل كل واحدة منهما عن اثنين وثلاثين محرفاً.

لا ترفع ملف البيئة إلى المستودع.

احتفظ بنسخة آمنة من القيمة:

```text
PIN_PEPPER
```

فقدان هذه القيمة يمنع التحقق من جميع الأرقام السرية المخزنة. لا تغيّرها إلا ضمن خطة لإعادة ضبط الأرقام السرية.

بعد ذلك شغّل:

```powershell
npm install
npm run migrate
npm start
```

افتح الموقع المحلي من رابط الخادم:

```text
http://127.0.0.1:3000
```

يمكن أيضاً فتح الواجهة من خادم معاينة محلي مثل المنفذ:

```text
5500
```

وستتصل تلقائياً بالخادم المحلي على المنفذ:

```text
3000
```

## استيراد البيانات التجريبية الحالية

الملف التالي مخصص للبيانات التجريبية فقط:

```text
data/clients.json
```

بعد تجهيز قاعدة البيانات، يمكن استيراده مرة واحدة:

```powershell
npm run import:clients
```

الأرقام السرية الموجودة داخله تتحول إلى بصمات عند الاستيراد.

لا تضع بيانات حقيقية أو أرقاماً سرية حقيقية داخل هذا الملف أو داخل المستودع.

## إنشاء حساب جديد

استخدم متغيرات مؤقتة، ثم شغّل أمر إنشاء الحساب:

```powershell
$env:NEW_ACCOUNT_NUMBER="3003"
$env:NEW_CLIENT_NAME="اسم العميل"
$env:NEW_CLIENT_PHONE="0790000000"
$env:NEW_CLIENT_PIN="73918462"
$env:NEW_CLIENT_BALANCE="0"
npm run create:client
Remove-Item Env:NEW_CLIENT_PIN
```

## النشر على خدمة الاستضافة

ملف النشر ينشئ خادماً وقاعدة بيانات بخطط مدفوعة، ويربطهما عبر الشبكة الخاصة:

```text
render.yaml
```

قيم الأسرار تنشأ تلقائياً أثناء النشر.

افتح الموقع من رابط الخادم نفسه. لا تستخدم نسخة منفصلة على خدمة صفحات ثابتة في الوضع الآمن، لأن الجلسة تعتمد على ملف ارتباط محمي ومقيد بالموقع نفسه.

الخطة المجانية لقاعدة البيانات غير مناسبة للبيانات الحقيقية؛ تنتهي بعد ثلاثين يوماً ولا توفر نسخاً احتياطية تلقائية.

## قبل إدخال بيانات حقيقية

- فعّل خطة قاعدة بيانات تحتوي على نسخ احتياطية واستعادة زمنية.
- اختبر استعادة نسخة احتياطية فعلياً.
- استخدم نطاقاً خاصاً وشهادة اتصال مشفر.
- لا تفعّل الإيداع الذاتي.
- غيّر جميع الأرقام السرية التجريبية.
- احذف أو عطّل الحسابات التجريبية.
- راقب سجل التدقيق والتنبيهات.
- نفّذ مراجعة أمنية مستقلة واختبار اختراق قبل الإطلاق العام.
